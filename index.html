CSS Layout Playground

This page is a cover/hero layout nested within a centered layout. A text center utility class was used to align the text center horizontally.

In this section

this is a combo of box, center, and stack. the box also has default variables declared which allow the bg and color to change. the box is providing the padding within the white, and the center is pushing everything to the center of the page. the content within the box is dictating the width, given the measure setting. the stack is providing spacing between the headers, paragraph content, and the next section.

switcher

until this point, the measure / center combo kept things a uniform width. with switcher, the flex basis/grow properties allow each element to grow the full width of the parent.

parent width

with no explicitly defined width, whis is where you would need some type of container class to define your max inline size

the switcher...

in this instance, allows each child to be displayed horizontally until min flex basis is reached, at which point each item collapses to the next block and takes up the full width

sidebar

sidebar content

this sidebar is set to have a width of 20 rem as a sidebar and will not grow or shrink until the main content breaches the set 50% threshold. after that, the main section will collapse beneath the sidebar.

this sidebar div also has stack applied to create vertical space between each element.

non-sidebar

non-sidebar content

the applied gutter value ensures there is space horizontally when the sidebar is in tact, as well as vertically when the layout shifts

these cards...

...are within a nested grid div on the main side of this sidebar layout. they still work responsively within this context
